## 3. PRIORITY FIXES (Ranked by PnL Impact)

### TIER 1: CRITICAL — Bot Must Trade (Days 1-3)

#### 1.1 Fix MIN_VOTES Death Spiral
**Problem**: With 3 strategies and MIN_VOTES=3, ALL strategies must agree. This is unanimity voting — nearly impossible in real markets.

**Options**:
- **Option A**: Enable monte_carlo_zones (`STRATEGY_MONTE_CARLO_ENABLED=true`) → 4 strategies, 3/4 = 75% consensus (recommended)
- **Option B**: Lower MIN_VOTES to 2 with safeguards (only for high-confidence signals)
- **Option C**: Both — 4 strategies, MIN_VOTES=2 for conf>80%, MIN_VOTES=3 for conf<80%

**Files**: `trading_config.py:119`, `ensemble.py:826`, `.env`

#### 1.2 Fix Regime "Unknown" Classification
**Problem**: Regime Agent classifies 94% as "unknown", triggering confidence penalties and preventing regime-specific strategies.

**Root Cause**: The Regime Agent prompt expects specific data patterns that may not match what's being sent. The snapshot may lack sufficient indicator context.

**Fix**: Ensure regime classification defaults to a usable regime (trend/range) based on ADX/EMA alignment rather than defaulting to "unknown" when uncertain. Add fallback logic in the strategy-level regime detection.

**Files**: `llm/agents/prompts.py` (regime prompt), `strategies/regime_trend.py`, `llm/agents/coordinator.py`

#### 1.3 Relax 6h Bearish Filter
**Problem**: confidence_scorer rejects ALL BUY signals when 6h MACD<0 AND MFI<45. In extended corrections, this kills every long signal for hours/days.

**Fix**: Either widen thresholds (MACD<-0.001 AND MFI<35) or downgrade to confidence penalty instead of hard rejection.

**Files**: `strategies/confidence_scorer.py:346-367`

#### 1.4 Remove LOSING_COMBOS Blacklist (or update it)
**Problem**: 3 hardcoded 2-strategy combos are auto-rejected, but with MIN_VOTES=3 this is redundant (2-agree can't pass anyway).

**Fix**: Remove blacklist if keeping MIN_VOTES=3, or keep it updated with real backtest data if lowering MIN_VOTES.

**Files**: `ensemble.py:836-848`

### TIER 2: HIGH — Bot Must Trade Smart (Days 3-7)

#### 2.1 Wire Growth System
**Problem**: Growth directory is empty. 7 subsystems initialized but producing zero output.

**Fix**: Debug `on_trade_closed()` flow → growth orchestrator. Verify file write permissions and data flow.

**Files**: `llm/growth/orchestrator.py`, `llm/growth/hypothesis_tracker.py`, `llm/growth/recommendation_engine.py`

#### 2.2 Enable Dynamic Strategy Weights
**Problem**: All weights stuck at 0.5 = no differentiation.

**Fix**: Make `get_rolling_weights()` the default, call `recompute_from_db()` every 10 trades.

**Files**: `data/strategy_weights.py`

#### 2.3 Fix Scout Token Limit
**Problem**: Scout agent response truncated at 768 tokens → incomplete JSON → lost watchlists.

**Fix**: Increase SCOUT max_tokens from 768 → 1024+ in `llm/agents/base.py:98`.

**Files**: `llm/agents/base.py`

#### 2.4 Evaluate LLM Mode
**Problem**: VETO_ONLY mode may be burning tokens without adding value if veto accuracy (vacc) < 0.50.

**Fix**: Track vacc metric. If < 0.50, switch to ADVISORY (log but don't enforce). Consider upgrading to SIZING if vacc > 0.60.

**Files**: `llm/autonomy_router.py`, `.env` (LLM_MODE)

### TIER 3: MEDIUM — Bot Must Learn (Days 7-14)

#### 3.1 Fix Memory Quality Gate
**Problem**: `_is_quality_note()` rejects most observations → 38 bytes of memory.

**Fix**: Lower minimum note length from 20 → 10 chars, allow more observation types.

**Files**: `llm/memory_store.py`

#### 3.2 Activate Curriculum Advancement
**Problem**: Stuck at Level 1 forever. No `curriculum_state.json` persisted.

**Files**: `llm/agents/coordinator.py`, `llm/growth/orchestrator.py`

#### 3.4 Fix Reconciliation

**Fix**: Pass wallet address in params OR ensure `HL_API_KEY`/`HL_API_SECRET` env vars are set.

**Files**: `execution/reconciliation.py:82`, `data/fetcher.py:278-284`, `.env`
