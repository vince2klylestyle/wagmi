================================================================================
                           CRITICAL ISSUES
================================================================================

[HIGH] Rounding-Induced Full Close (Lines 549-551)
   Issue: If close_qty rounds to full qty, partial close silently becomes full close
   Impact: User expects 50% partial but gets 100% full close
   Fix: Add warning log before converting to TP1_FULL
   Recommendation: Add log like "TP1 partial close: close_qty rounded to full"

[HIGH] Negative Profit Cushion After TP1 Loss (Lines 574-588)
   Issue: If TP1 closes at loss, profit_cushion becomes negative
   Impact: New SL can move ABOVE entry (correct math but risky)
   Fix: Add validation check for unreasonable SL inversion
   Recommendation: Log warning if new SL > (entry + 50% of original risk)

[MEDIUM] Funding Cost Sanity Check Missing
   Issue: No validation that funding_costs stay reasonable
   Impact: Mis-tracking could distort final PnL significantly
   Fix: Add check before final close: if funding > realized_pnl*1.5, warn
   Recommendation: Validate during position close

[MEDIUM] Regime Detection Stale
   Issue: Early exit uses entry_reasons["regime"] which was set at entry
   Impact: May not adapt to regime changes during hold
   Fix: Compute current regime from fresh technical indicators
   Recommendation: Add _get_current_regime() method

[MEDIUM] Hard-Coded Trailing Multipliers
   Issue: Style multipliers (0.8/1.0/1.5) not tunable
   Impact: Can't adapt to different volatility environments
   Fix: Make configurable via env vars
   Recommendation: Add TRAILING_MULT_TIGHT, TRAILING_MULT_MEDIUM, TRAILING_MULT_LOOSE
